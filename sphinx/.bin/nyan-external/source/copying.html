Per-file license header guidelines:

In addition to this file, to prevent legal caveats, every source file must include a header.

nyan-native source files, that is, files that were created by the nyan authors, require the following one-line header, preferrably in the first line, as a comment:

Copyright 20XX-20YY the nyan authors, LGPLv3+. See copying.md for legal info.

20XX is the year when the file was created, and 20YY is the year when the file was last edited. When editing a file, make sure the last-modification year is still correct.

This file originates from openage. It was adapted for nyan and its license.

I (zuntrax) and neither mic_e are lawyers. This is an open-source project, we’re doing this for fun. People convinced me that this legal shit must be done, so I did it, even though I’d rather have spent the time on useful parts of the project. If you see any legal issues, feel free to contact me. I, personally, despise in-sourcefile legal text blocks. They’re a pest, and unlike many others, I don’t simply accept them because “that is what everybody does”. Thus, I worked out the minimal 1-line text above, which should be free of legal caveats, and a reasonable compromise. I’d be happy to see it used in other projects; you’re free to use this file (copying.md) as a template for your project’s legal documentation.
